## 新旧代码隔离检查清单

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 新功能是否有独立目录/包 | □ | 便于单独测试和审查 |
| 是否通过接口与旧代码交互 | □ | 降低耦合 |
| 是否需要修改已有接口 | □ | 如需修改，列出所有影响点 |
| 新功能是否有向后兼容要求 | □ | API/数据格式兼容 |
| 是否需要版本迁移逻辑 | □ | 数据库、配置等迁移 |

## 快速启动命令

```bash
# 0. 构建 knowledge 工具
cd tools/knowledge
go build -o knowledge.exe .
cd ../..

# 0a. 初始化知识库（首次使用）
knowledge init

# 1. 检查知识库状态
ls -la project-knowledge/INDEX.md 2>/dev/null || echo "知识库不存在，将首次创建"

# 1. 搜索知识库（检索可复用代码）
knowledge search <功能关键词>

# 2. 检查知识条目是否过期
knowledge check-stale .

# 3. 创建新模块目录
mkdir -p internal/features/<feature_name>

# 4. 初始化 Go 模块（如需）
go mod init your/module

# 5. 新功能代码结构示例
internal/features/
├── myfeature/
│   ├── myfeature.go       # 主逻辑
│   ├── myfeature_test.go   # 测试
│   ├── mock_*.go          # Mock（如需要）
│   └── doc.go             # 包级文档

# 6. 编译测试
go build ./internal/features/myfeature
go test -race -cover ./internal/features/myfeature

# 7. 静态检查
go vet ./internal/features/myfeature
gofmt -l ./internal/features/myfeature

# 8. 保存知识条目
knowledge add -t "[标签] 功能名 - 用途" -s "internal/features/myfeature/" -p "用途描述"
```

## 问题定义要素表

在开始编码之前，必须澄清以下要素：

| 要素 | 问题 | 示例 |
|------|------|------|
| 目的 | 解决谁的什么问题？ | 解决运维人员批量管理服务器的问题 |
| 场景 | 在什么情况下触发？ | 在凌晨批量执行部署任务时 |
| 结构 | 输入→处理→输出？ | 服务器列表→并行执行→结果汇总 |
| 约束 | 性能、安全、技术边界？ | 最多 100 台，支持 SSH key 认证 |
| 验收 | 怎么算"完成"？ | 所有服务器部署成功，返回详细日志 |

**如果需求不清晰，优先向用户提出不超过 3 个澄清问题。**

## 模块拆分原则

将大功能拆为独立的、可验证的微模块：

```
Feature/
├── module_1/
│   ├── module_1.go       # 业务逻辑
│   └── module_1_test.go  # 单元测试
├── module_2/
│   ├── module_2.go
│   └── module_2_test.go
└── integration_test.go   # 集成测试
```

**每个模块原则**：
- 单一职责（SRP）
- 有明确的输入输出
- 可独立验证
- 最小依赖

## 代码生成循环

循环执行以下步骤：

```
┌─────────────────────────┐
│  1. 设计思路 + 数据结构  │
│     （等用户确认）        │
├─────────────────────────┤
│  2. 生成完整代码         │
│     （含错误处理）        │
├─────────────────────────┤
│  3. 生成单元测试          │
│     （含边界测试）        │
├─────────────────────────┤
│  4. 验证测试通过          │
│     （go test）          │
└─────────────────────────┘
         ↓
    下一个模块
```

## 集成验证命令

```bash
# 1. 静态检查
go vet ./...
gofmt -l .

# 2. 单元测试 + 覆盖率
go test -race -cover ./...

# 3. 集成测试（如有）
go test -tags=integration ./...

# 4. 构建验证
go build -o bin/app ./cmd/app

# 5. 知识沉淀：保存新增的可复用函数/模式
knowledge add -t "[模式] 本次新增模式名" -s "pkg/xxx/xxx.go" -p "用途描述"
knowledge list  # 确认条目已更新
```
