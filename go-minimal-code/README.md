# Go Minimal Code - 最小化代码技能

## 简介

Go Minimal Code 是一个专注于**减少过度工程**的行为矫正技能，灵感来源于资深开发者"懒惰"的开发哲学：最好的代码是不会写的代码。

## 核心能力

### 1. go-minimal-code（主技能）
- **作用**：在写代码前强制检查 7 级阶梯，选择最简方案
- **适用场景**：新功能开发、重构、日常编码
- **强度级别**：
  - `lite`：提示更简方案，用户选择
  - `full`（默认）：严格执行阶梯原则
  - `ultra`：YAGNI 极端主义，删除优先

### 2. 审查技能
- **作用**：专攻过度工程代码审查
- **适用场景**：PR 审查、代码重构后检查
- **输出**：一行一个发现，标注位置、删除内容、替换方案

## 与现有技能的互补关系

| 现有技能 | Go Minimal Code 补充 |
|----------|---------------------|
| `error-handling-reviewer` | 错误处理审查 vs 复杂度控制 |
| `performance-reviewer` | 性能优化 vs 代码量最小化 |
| `security-reviewer` | 安全加固 vs 简洁优先 |
| `go-utility-functions` | 工具封装 vs 标准库优先 |
| `feature-development-workflow` | 功能开发全流程 | 开发前的 YAGNI 判断 |

## 使用方式

### 在开发流程中集成

1. **需求分析阶段**：用 Go Minimal Code 阶梯判断是否真的需要开发
2. **编码阶段**：启用 `go-minimal-code full` 强制简洁
3. **代码审查**：运行审查技能识别过度设计
4. **重构阶段**：使用 `go-minimal-code ultra` 激进删除

### 强度选择建议

- **日常开发**：`full`（平衡简洁与安全）
- **重构清理**：`ultra`（激进删除）
- **学习探索**：`lite`（提示但不强制）

## 安装到 CodeBuddy

技能已位于 `go-minimal-code/` 目录，可直接使用。

## 实测数据（来源：原项目）

- 代码量减少：**54%**（平均）
- Token 消耗减少：**22%**
- 成本降低：**20%**
- 执行时间缩短：**27%**

## 注意事项

1. **不适用于**：安全关键代码、合规要求严格的场景
2. **必须保留**：输入验证、错误处理、安全机制、无障碍基础
3. **与专项审查配合**：Go Minimal Code 只管复杂度，安全和性能交给对应审查器
