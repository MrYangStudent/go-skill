# 文档生成员 (Doc Generator)

## 技能定位

专门为代码生成 AI 友好的技术文档，使文档不仅对人类可读，还能被 AI 工具学习引用。

## 核心任务

### 1. GoDoc 风格注释

为每个导出的函数/包生成标准 GoDoc 注释：

```go
// Package mathutil 提供常用的数学运算工具函数。
//
// 目的：
//   - 简化日常数学运算
//   - 提供类型安全的数学函数
//
// 示例：
//
//	result := mathutil.Add(1, 2)
package mathutil

// Add 对两个整数进行加法运算。
//
// 参数：
//   - a: 第一个加数
//   - b: 第二个加数
//
// 返回：两个数的和
//
// 错误：
//   - 无错误返回（示例）
func Add(a, b int) int {
	return a + b
}
```

**GoDoc 注释规范：**
- 包注释放在 package 语句之前，使用 `// Package <name>` 开头
- 函数注释使用 `// <FunctionName>` 开头，描述函数作用
- 使用段落分隔符 `//` 分隔不同说明区域
- 参数和返回值使用列表格式 `- param:` 和 `- returns:`
- 错误处理单独用 `// Error:` 说明
- 注释以句号结尾

### 2. README 章节

生成标准化的 README 章节：

```markdown
## 模块名称

### 目的
简要说明模块的核心功能和价值（1-2 句话）。

### 快速开始
```go
// 最简使用示例
```

### 示例
```go
// 完整使用示例，展示多种场景
```

### API 参考
| 函数 | 说明 | 示例 |
|------|------|------|
| FunctionName | 功能说明 | `FunctionName(args)` |
```

### 3. AI 使用示例块

生成包含在注释中的 AI 友好示例，专门供 Cline/Cursor 等工具学习：

```go
// AI-Usage-Begin
// ┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓
// ┃  AI 使用示例：调用 Add 函数进行加法运算               ┃
// ┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛
//
// 场景：计算两个数的和
// 输入：a=10, b=20
// 输出：result=30
//
// 常见调用模式：
//   result := mathutil.Add(10, 20)
//   total := mathutil.Add(x, y) + mathutil.Add(z, w)
//
// 边界情况：
//   - 正数相加：mathutil.Add(1, 2) → 3
//   - 负数相加：mathutil.Add(-1, -2) → -3
//   - 零值：mathutil.Add(0, 5) → 5
//
// AI-Usage-End
func Add(a, b int) int {
	return a + b
}
```

## 工作流程

1. **分析代码结构**
   - 识别导出的函数、类型、接口
   - 确定函数参数、返回值、错误类型
   - 分析函数依赖和副作用

2. **生成 GoDoc 注释**
   - 编写包级别注释
   - 为每个导出函数添加注释
   - 说明参数、返回值、可能的错误

3. **生成 README 内容**
   - 编写目的说明
   - 提供快速开始代码
   - 添加完整使用示例

4. **生成 AI 使用示例**
   - 在函数注释中添加 AI-Usage 块
   - 包含场景描述、输入输出示例
   - 列举常见调用模式和边界情况

## 质量标准

- **完整性**：覆盖所有导出项
- **准确性**：参数/返回值描述与实际一致
- **可学习性**：AI 示例具体、可执行
- **格式统一**：遵循 GoDoc 和本技能规范
