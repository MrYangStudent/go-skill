# 进阶提示词技术

> 吴恩达课程之外的延伸技术，进一步提升提示词的精确度和能力上限

---

## 1. 思维树（Tree-of-Thought, ToT）

### 概念
让 AI 在推理过程中同时探索多个可能路径，而非像 CoT 只走一条直线。AI 先生成多个思路分支，然后评估每条分支的可行性，选择最优路径继续深入。

### 与 CoT 的区别

| 维度 | Chain-of-Thought | Tree-of-Thought |
|------|-----------------|-----------------|
| 探索方式 | 单一路径线性推理 | 多分支并行探索 |
| 分支评估 | 无 | 每条分支评估可行性 |
| 适用场景 | 有标准答案的逻辑题 | 需要创造性探索的开放问题 |
| 输出长度 | 中等 | 较长 |

### 通用 Prompt 模板

```
请用"思维树"方法分析以下问题。

第一步：生成 [3-5] 个不同的解决思路/分析角度，每个用一句话概括。

第二步：评估每个思路的可行性和潜在价值（高/中/低），并简要说明理由。

第三步：选择最有潜力的 [1-2] 个思路，深入展开具体方案。

第四步：检查被放弃的思路中是否有可吸取的部分，补充到最终方案中。

问题：[在此处输入你的问题]
```

### 适用场景
- 商业策略制定
- 产品功能设计
- 复杂问题分析（无明显标准答案）
- 创意方案生成

## 2. ReAct 模式（Reasoning + Acting）

### 概念
ReAct 让 AI **交替进行"推理"和"行动"**：推理当前状态 → 决定下一步行动 → 执行观察结果 → 继续推理。这是 AI Agent（智能体）的底层思维模式。

### 适用提示词结构

```
请按照 ReAct 模式完成任务：

**思考（Thought）**：分析当前状态，我需要做什么？
**行动（Action）**：我需要执行什么操作（搜索、计算、查询等）？
**观察（Observation）**：行动后获得的结果是什么？

重复以上循环，直到任务完成，然后输出最终答案。

任务：[描述任务]

如果某些操作无法实际执行（如无法联网），请在"行动"步骤说明假设的操作，
并在"观察"步骤基于已有知识推理可能的观察结果。
```

### 应用示例：市场调研

```
思考：我需要了解 2024 年国内 AI 写作工具的市场规模和主要玩家。
行动：搜索 2024 年国内 AI 写作工具市场报告
观察：根据训练数据，2024 年市场规模约 XX 亿元，主要玩家有 A、B、C...

思考：已经有市场规模数据，现在需要竞品的功能差异对比。
行动：对比 A、B、C 三款工具的核心功能
观察：A 专注学术写作，B 专注营销文案，C 是做全品类...

思考：信息收集完成，可以输出市场分析报告了。
```

### 适用场景
- 需要多步推理的研究任务
- 信息收集与综合分析
- 需要外部工具配合的复合任务

## 3. 提示词链接（Prompt Chaining）

### 概念
将一个大任务拆解为多个**顺序执行的子提示词**，每个子提示词的输出作为下一个的输入。相比"一步到位"，链接式提示词允许在中间步骤进行人工检查和调整。

### 通用框架

```
# 链接 1：分析输入
[输入原始材料，让 AI 先做摘要和分析]

# 人工检查摘要是否准确

# 链接 2：生成大纲
[基于摘要生成结构大纲]

# 人工确认大纲方向

# 链接 3：撰写正文
[基于大纲逐段生成正文]

# 人工逐段审核

# 链接 4：润色校对
[让 AI 检查语法、逻辑、风格一致性]
```

### 与"一步到位"的对比

| 方面 | 一步到位 | 提示词链接 |
|------|---------|-----------|
| 控制粒度 | 只有一次控制机会 | 每个环节都可干预 |
| 修改成本 | 改一点等于重做 | 可在中间步骤修正方向 |
| 输出一致性 | 内部逻辑可能矛盾 | 后续步骤可引用前面结果 |
| 适用场景 | 简单、明确的任务 | 复杂、需要迭代的任务 |

### 实现方式

**方式一：多轮对话（推荐）**
```
用户(第1轮): 请先分析以下数据，告诉我关键趋势...
用户(第2轮): 基于你的分析，写一份报告大纲...
用户(第3轮): 按照大纲完成正文...
```

**方式二：结构化提示词**
```
请按以下步骤执行，在没有完成上一步前不要进入下一步：
步骤1: 分析输入数据，提取关键趋势
步骤2: 基于趋势生成报告大纲
步骤3: 按照大纲撰写正文

如果前一步的产出质量不够好，AI 应该主动要求用户确认后再继续。
```

## 4. AI 智能体提示模式（Agentic Prompting）

### 概念
不给 AI 具体操作指令，而是给它**一个目标、一组可用工具、一套行为规则**，让 AI 自主规划如何达成目标。

### 核心要素

```
目标：[描述最终要达成的结果]
可用工具：[列出 AI 可以使用的工具/能力]
约束条件：[时间、资源、质量等限制]
行为规则：
1. [如：每次行动前先说明计划]
2. [如：遇到不确定时主动询问]
3. [如：输出最终结果前需做自我检查]
```

### 示例：研究助理 Agent

```
目标：撰写一份关于"2024 年全球电动汽车市场趋势"的研究简报，不超过 5 页。

可用工具：
- 网络搜索（查询最新数据）
- 深度研究（综合多来源分析）
- 代码运行（数据分析与可视化）

约束条件：
- 数据来源需标注（公司名/报告名/年份）
- 优先使用 2023-2024 年的数据

行为规则：
1. 先制定研究计划，等我确认后再执行
2. 数据不确定时标注"待核实"，不要编造
3. 最终输出前做一次事实核查
4. 用中文输出，关键术语保留英文原名
```

### 适用场景
- 开放式研究任务
- 多步骤数据分析
- 内容创作与编辑

## 5. 元提示（Meta-Prompting）

### 概念
让 AI **自己设计提示词**。告诉 AI 你想要达成的目标，让 AI 先生成一个针对该目标的优化提示词，用户使用该提示词再与 AI 交互。

### 通用模板

```
我想完成以下任务：[描述任务目标]
请帮我设计一个完整、结构化的提示词，发送给 AI 后可以获得高质量的[输出内容]。

你的提示词需要包含：
1. 角色定义（AI 应该扮演什么角色）
2. 背景说明（提供必要的上下文）
3. 明确的输出要求（格式、长度、风格）
4. 约束条件（避免什么，注意什么）
5. 评估标准（如何判断输出是否合格）

提示词要可直接复制使用。
```

### 元提示 vs 手动构建

| 方式 | 适合人群 | 优缺点 |
|------|---------|--------|
| 手动六步构建 | 初学者 | 学习效果好，理解每个维度 |
| 元提示 | 有经验的用户 | 效率高，但可能错过某些维度 |

## 6. 提示词压缩（Prompt Compression）

### 概念
当上下文窗口有限或需要节省 token 时，将冗长的提示词压缩为更紧凑的格式，同时保留核心指令。

### 压缩技巧

**技巧 1：缩写约束**
```
# ❌ 冗长版
请不要使用复杂的专业术语，如果要使用请先解释。
文章风格要轻松易懂，适合没有技术背景的普通读者。
段落不要太长，每段不超过 5 句话。
不要用"随着时代的发展"这样的套话。

# ✅ 压缩版
风格：轻松科普级，面向零背景读者
约束：禁用术语（必须用则解释）| 每段≤5句 | 禁用套话模板
```

**技巧 2：结构化字段**
```
# ❌ 冗长版
我的目标受众是 25-35 岁的职场人士，他们通常工作繁忙，希望提高工作效率。
我需要的是一篇关于时间管理的实用文章，最好能提供具体的方法和步骤。
文章长度在 1500 字左右。
请用中文写作。

# ✅ 压缩版
受众：25-35岁职场人（工作忙、求实用）
主题：时间管理实操方法
字数：1500字
语言：中文
```

**技巧 3：符号分隔**
```
角色: 生产力教练
受众: 25-35岁职场人 | 痛点: 时间不够用
目标: 提供可落地的3个时间管理方法
约束: 每个方法必须有步骤说明 | 1200-1800字
格式: 痛点切入→分方法讲解→行动计划
```

### 适用场景
- API 调用（上下文窗口有限）
- 长文档处理的 prompt 部分
- 快速迭代测试提示词

## 7. 前置条件式硬约束（Precondition Hard-Constraint）

### 概念
将提示词中的约束条件从"软性建议"转化为**硬性前置条件**——AI 必须在开始执行前确认这些约束，并在执行过程中严格遵守。

### 与普通约束的区别

| 方面 | 软约束（普通写法） | 硬约束（前置条件式） |
|------|-------------------|---------------------|
| 表态方式 | "最好..." / "尽量..." | "必须..." / "禁止..." |
| 检查机制 | 依赖 AI 自觉 | 要求 AI 在输出前自我验证 |
| 确定性 | AI 可能遗漏 | 每一步强制执行 |

### 实现方法

**方法 1：前置声明**
```
在开始之前，请确认以下硬约束：
1. [约束 1]  2. [约束 2]  3. [约束 3]
如果输出违反以上任何约束，立即停止并重新生成。
```

**方法 2：后置验证**
```
在输出最终结果前，逐一检查以下条件：
✅ [条件 1]  ✅ [条件 2]
不满足则修正后再输出。
```

**方法 3：禁忌列表**
```
以下行为被严格禁止：🚫 [禁止行为 1]  🚫 [禁止行为 2]
如需触发，暂停并说明替代方案。
```

### 四维优化

1. **清晰度**：消除歧义，使约束意图明确、单义
2. **专业度**：使用规范、严谨的工程化语言
3. **完备性**：覆盖所有可能偏离的场景，不留灰色地带
4. **可验证性**：每条约束都可被 AI 自我检查

### 适用场景
- 金融/医疗等高安全性代码生成
- 严格格式要求的文档生成
- 防止 AI 编造数据或引用
- 需要精确遵守业务规范的任务

## 技术选型指南

| 问题类型 | 推荐技术 |
|----------|---------|
| 有标准答案的推理题 | CoT（思维链） |
| 开放式创意探索 | ToT（思维树） |
| 信息收集+综合分析 | ReAct 模式 |
| 复杂内容创作 | 提示词链接 |
| 开放目标自主完成 | Agentic Prompting |
| 想快速获得好提示词 | 元提示 |
| 上下文窗口紧张 | 提示词压缩 |
| 需要严格约束AI行为 | 前置条件式硬约束 |
