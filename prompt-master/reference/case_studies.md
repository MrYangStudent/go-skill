# 提示词优化前后对比案例

> 真实场景下的提示词优化案例，展示六维诊断如何提升输出质量

---

## 案例 1：通用文章写作

### ❌ 原始提示词

```
帮我写一篇关于远程办公的文章。
```

**诊断结果**：角色❌ 背景❌ 目标⚠️ 约束❌ 格式❌ 示例❌

**AI 的典型输出问题**：
- 泛泛而谈，缺乏具体视角
- 结构模板化（背景→现状→展望）
- 没有具体数据和个人观点
- 读完没有记忆点

---

### ✅ 优化后提示词

```
角色：你是一位有 10 年经验的远程团队管理顾问，曾为多家 500 强企业设计远程工作制度。

背景：需要为一本面向企业管理者的杂志撰写文章。读者正在考虑或已经实施远程办公，关注实操方法而非理论。

目标：写一篇 2000 字左右的文章，主题为"远程办公的隐藏成本及应对策略"，让管理者读完后能识别出自己团队可能忽略的 3 个远程办公陷阱，并获得可落地的解决方案。

约束：
- 每个论点必须有真实数据或案例支撑
- 不讨论远程办公的基础定义和常识
- 语言风格：专业务实，不夸张
- 禁用"随着时代的发展""在当今社会"等套话
- 至少包含 1 个来自知名公司的真实案例

输出格式：
1. 引人注目的标题（10-15 字，含核心观点）
2. 引言：一个真实故事或数据切入
3. 正文：3 个隐藏成本，每个包含"问题→数据→案例→解决方案"
4. 总结：可行动的 checklist
```

**改进效果**：
- 从"泛泛科普"变为"有深度的管理者参考"
- 输出结构明确，不再需要后期大量调整
- 内容有数据有案例，可信度大幅提升

## 案例 2：代码生成

### ❌ 原始提示词

```
帮我写一个 REST API。
```

**AI 的典型输出**：
- 用最通用的技术栈（Express.js / Flask）
- 没有错误处理
- 没有输入验证
- 没有注释

---

### ✅ 优化后提示词

```
角色：你是一位精通 Go 语言的后端高级工程师，熟悉 Clean Architecture 和 RESTful 设计规范。

背景：正在开发一个电商平台的后端服务。需要实现用户注册/登录的 API，作为项目的一部分。

项目技术栈：
- 语言：Go 1.22+
- 框架：标准库 + chi router
- 数据库：PostgreSQL
- 认证：JWT token
- 仅使用标准库和已声明的依赖，无额外第三方包

目标：生成以下三个 API endpoint 的完整代码：
1. POST /api/v1/register — 用户注册（用户名、邮箱、密码）
2. POST /api/v1/login — 用户登录，返回 JWT token
3. GET /api/v1/profile — 获取当前用户信息（需认证）

约束：
- 输入参数需做完整验证（格式、长度、空值检查）
- 错误码遵循 HTTP 语义（400/401/403/404/500）
- 密码使用 bcrypt 加密
- JWT 有效期 24 小时
- 所有函数必须包含注释
- 包含数据库迁移 SQL

输出格式：
1. 每个 endpoint 的 handler 代码
2. 中间件代码（JWT 验证）
3. 数据库模型和迁移 SQL
4. 使用说明（如何运行和测试）
```

**改进效果**：
- 技术栈精确定位，不再需要手动替换框架
- 包含了实际项目需要的完整工程规范
- 输出直接可用，无需再补充错误处理、验证等

## 案例 3：头脑风暴

### ❌ 原始提示词

```
帮我想一些营销创意。
```

**AI 的典型输出**：
- 安全、常规的营销手段（社交媒体、KOL 合作、打折促销）
- 缺乏针对性和创新性
- 没有考虑预算和资源约束

---

### ✅ 优化后提示词

```
角色：你是一位增长黑客，擅长在预算有限的情况下找到高 ROI 的营销杠杆。

背景：
- 产品：一款面向大学生的 AI 写作助手 App（已上线）
- 当前困境：用户获取成本高，口碑传播慢
- 预算：月营销预算 5000 元
- 团队：3 人（我 + 1 设计 + 1 运营）
- 已有渠道：小红书、校园大使（10 人）

目标：找到 3 个可在 2 周内启动的低成本营销方案，预期能带来至少 2000 新注册用户。

约束：
- 每个方案的总执行成本不超过 2000 元
- 不依赖付费广告投放
- 关注大学生群体的真实触达渠道
- 方案必须可量化效果

要求：
1. 从以下三个角度分别输出方案：病毒传播、校园渗透、内容营销
2. 每个方案标注：预估成本、预期效果、执行时间线、关键风险
3. 最后用 ★ 标注实用性和创新性
4. 对每个方案做一个"魔鬼代言人"质疑
```

**改进效果**：
- 从"空泛的营销点子"变为"可执行的落地计划"
- 考虑了预算、团队、时间的真实限制
- 有风险评估，避免想当然

## 案例 4：AI 评审（反迎合示范）

### ❌ 有偏见的提问

```
我写了一份非常棒的产品方案，帮我看看有什么问题。
```

**隐含问题**：你说了"非常棒"，AI 会倾向于说好话而非真实评价。

**AI 的可能回应**：
- "您的方案思路清晰，考虑周全..."
- 轻描淡写地提一两个小问题，然后再次肯定

---

### ✅ 反迎合提问

```
角色：你是一位有 10 年经验的产品评审专家，以严格、客观著称。你需要给我最真实的反馈，不要顾及我的感受。

背景：以下是我为[产品名称]撰写的产品方案。

评审维度：
1. 用户痛点 — 需求是否真实？足够痛？
2. 解决方案 — 方案是否真能解决问题？
3. 可行性与风险 — 技术、资源、时间风险

要求：
1. 先逐项找问题：每个问题必须指出具体位置和原因
2. 在所有问题列出后再给出综合评价
3. 不要一开始就打分，先列证据
4. 标注：哪些是确定的问题，哪些是你推测的
5. 如果方案确实好，也请说明好在哪里

特别注意：
- 如果方案有问题，请直接说"这个方案有问题"，不要委婉
- 不要因为我说了"非常棒"就降低标准

---
【方案内容】
...
```

**改进效果**：
- 从"安全肯定"变为"有实质价值的评审"
- 评审结果可执行，知道具体哪里需要改
- 避免了 AI 的迎合性偏差

## 案例 5：写作工作流（先结构后正文）

### ❌ 一步到位

```
帮我写一篇关于时间管理技巧的文章。
```

**问题**：AI 直接生成全文后，发现结构不对想改？改结构等于重写。

---

### ✅ 分步工作流

```
=== 阶段一：大纲 ===

角色：你是一位生产力教练，帮职场人士提升效率。

背景：为一篇面向 25-35 岁职场人士的知识文章准备。
目标读者痛点：每天感觉时间不够用，尝试过很多方法但坚持不下来。

请提供 3 种不同结构的大纲方案：
1. "问题→方法"式 — 针对不同时间管理问题给对应方案
2. "案例驱动"式 — 用一个职场新人的成长故事串联方法
3. "观念刷新"式 — 先打破常见时间管理误区，再给新思路

每种大纲标注：目标读者感受、核心论点、段落安排。


=== 在用户确认大纲后 ===

阶段二：在选定的大纲小标题下补充关键要点（每条 2-3 个点）
阶段三：大纲和要点确认后，才生成正文草稿
```

**改进效果**：
- 结构在动笔前就已经定好
- 修改成本大幅降低（改大纲 vs 改全文）
- 最终文章逻辑更连贯，AI 腔更少
