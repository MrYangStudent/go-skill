# Prompt Master - 提示词大师

> 基于吴恩达《AI Prompting for Everyone》课程精华的对话式提示词构建引擎

## 概述

本技能通过**结构化对话引导**，帮助用户从零开始构建高质量提示词，或对现有提示词进行诊断和优化。核心采用"六步工作流"，每一步收集一个关键维度的信息，最终生成结构完整、目的明确的优质提示词。

## 核心原则

1. **结构化优于自由发挥** — 按照六步框架构建的提示词覆盖更全面、产出更可控
2. **以终为始** — 先明确输出目标和格式，再反推提示词的内容组织
3. **具体优于抽象** — 具体示例和详细约束比笼统描述更有效
4. **迭代优化** — 好的提示词是迭代出来的，每次使用后分析不足并改进

## 关键认知（来自课程核心洞察）

### AI 新手 vs 高级用户的四个差距

| 维度 | AI 新手 | AI 高级用户 |
|------|---------|-------------|
| 提问深度 | 简单搜索式提问（"双层塔可还有吗？"） | 上传多份文档要求深度分析（"对比这些保险方案"） |
| 提供上下文 | 简短提示，期待 AI 猜你想要什么 | 如数上传相关资料，设身处地为 AI 着想 |
| 获取诚实反馈 | 带偏见提问（"我很棒的主意，评价一下"）→ 收获谄媚 | 中立提问或给出评分标准，迫使 AI 客观 |
| 写作流程 | 直接让 AI 写全文 → 得到"AI 腔"烂泥 | 先出大纲 → 批评迭代 → 最终成文 |

**核心心态转变**：把 AI 当作"分析师"而非"搜索引擎"。你不是问一个问题，而是委托一项任务。问题期待一个答案，任务期待一份交付物。

### 信息获取的三种方式对比

| 方式 | 来源数量 | 时效性 | 响应时间 | 适用场景 |
|------|----------|--------|----------|----------|
| 预训练知识 | 模型内化 | 截止日期前 | 几秒 | 常识、经典理论、稳定知识 |
| 普通网络搜索 | 少量来源 | 较新 | 数秒~数十秒 | 实时信息、位置、小众信息 |
| 深度研究 | 数十个来源 | 最新 | 数分钟 | 复杂问题、多来源综合分析 |

**技巧**：在提示词中加入年份或时间词（"2025年"）能强制 AI 触发网络搜索；对需要多角度权衡的复杂问题，优先使用深度研究模式。

### 预训练知识的边界

- **知识频率决定可靠性**：话题在互联网越常见，AI 回答越可靠（常识>冷门知识）
- **拼写容忍度高**：不需要过度纠正语法错误，AI 能理解错别字
- **关键局限**：不含私有数据、不含截止日期后的信息、互联网上的错误也会被学到
- **"摘要陷阱"**（Source Summary Trap）：AI 进行网络搜索时并未完整阅读网页，只看摘要片段就生成回答。重要信息必须点开原始链接核实。

## 对话式提示词构建工作流

### 第一步：角色定义（Role）

**目标**：确定 AI 在任务中扮演的角色/身份，定调语气和专业度。

**引导问题**：
- "你希望 AI 扮演什么角色来帮助你？"
- 示例角色：专业写作顾问、资深程序员、数据分析师、教学导师、翻译专家、心理咨询师、营销策划师等

**输出影响**：
- 角色越明确，AI 的输出语气和专业度越精准
- 复合角色声明效果更佳，例如："你是一位有 10 年经验的前端开发工程师"比"你是一位程序员"好

**常见错误**：
- ❌ 角色过于宽泛（如"你是助手"）
- ❌ 角色与任务不匹配

### 第二步：背景描述（Context）

**目标**：提供任务的上下文信息和前置条件，让 AI 理解场景全貌。

**引导问题**：
- "这个任务在什么场景下进行？"
- "目标受众是谁？"
- "有什么前置条件或已知信息？"

**输出影响**：
- 背景越充分，AI 越能给出贴合实际的回答
- 包含受众信息（如面向初中生 vs. 面向博士生）会显著影响输出风格

### 第三步：目的阐述（Objective）

**目标**：明确具体期望达成的目标和产出方向。

**引导问题**：
- "你希望最终得到什么？"
- "这个提示词要解决什么问题？"
- "成功的标准是什么？"

**输出影响**：
- 目的是提示词的核心驱动力
- 目的越具体、可衡量越好

### 第四步：约束设定（Constraints）

**目标**：定义限制条件、边界规则和避坑事项。

**引导问题**：
- "有什么限制条件？"
- "需要避免什么？"
- "有什么必须遵守的规则？"

**约束类型**：
| 类型 | 示例 |
|------|------|
| 长度限制 | "不超过 500 字" |
| 风格限制 | "正式学术风格，不使用口语化表达" |
| 内容排除 | "不涉及价格信息，不讨论政治" |
| 技术要求 | "必须使用 Python 3.10+，只使用标准库" |

### 第五步：输出格式（Output Format）

**目标**：指定结果以何种结构呈现。

**引导问题**：
- "你希望结果以什么形式呈现？"
- "用列表、表格、代码块还是纯文本？"
- "需要分几个部分？"

**格式选项**：
| 格式 | 适用场景 |
|------|----------|
| 段落文本 | 说明、分析、建议 |
| 列表 | 步骤、要点、检查项 |
| 表格 | 对比、数据汇总 |
| 代码块 | 编程、配置文件 |
| Markdown | 结构化文档 |
| JSON | 机器可读数据 |

### 第六步：示例引导（Example - 可选）

**目标**：提供 1-3 个参考示例，明确风格和质量的预期标准。

**引导问题**：
- "有没有参考示例？"
- "可以参考什么风格或案例？"
- "有没有希望达到的类似输出？"

**技术价值**：
- 示例引导是最强大的提示词技术之一（即少样本学习）
- 好的示例可以大幅提高输出质量
- 包括"好示例"和"差示例"的对比尤为有效

## 头脑风暴工作流（Brainstorming with AI）

> 来自课程 Module 2 核心内容。AI 是绝佳的"思考伙伴"，但需要正确的引导方式。

### 核心原则

1. **提供充足背景** — 不要只给一句话，要说明你的领域、目标、已有哪些想法、想探索的方向
2. **追求多样输出** — 让 AI 从不同角度生成多个可能性，不要只满足于第一个答案
3. **迭代深化** — 先发散再聚焦：第一轮广撒网，第二轮深入筛选后的方向
4. **人机互补** — AI 擅长广度和联想，人类擅长判断和决策

### 引导模板

```
请帮我进行头脑风暴。以下是背景信息：

【领域/主题】[你的领域]
【目标】[想达成什么]
【已有哪些想法】[列出已想到的点]
【探索方向】[想从哪些方向突破]

要求：
1. 先从 3-5 个不同角度提出想法
2. 每个想法简要说明其可行性和创新点
3. 最后将想法标注为：实用性 ★~★★★ / 创新性 ★~★★★ / 可执行性 ★~★★★
4. 标注你认为最有潜力的 2 个方向及理由
```

### 进阶头脑风暴技巧

- **限制激发创意**：给 AI 反向约束（"预算只有 100 元""只能用 3 个人完成"）反而能产生更有创意的方案
- **角色切换**：让 AI 依次扮演不同角色（营销总监、技术专家、用户）输出多视角方案
- **极端假设**：要求 AI 完全推翻已有假设，从零开始构想
- **"魔鬼代言人"**：对每个想法要求 AI 从反对角度列出问题，避免集体盲区

## 进阶提示词技巧

### 角色扮演（Role-Playing）
- 让 AI 扮演特定角色可以获得专业知识背景下的精准回答
- 原则：越具体越好，包含背景、专业领域、经验年限
- 示例："你是一位在 Google 工作 8 年的 UX 研究员"

### 思维链（Chain-of-Thought, CoT）
- 引导 AI 一步步思考，更适合复杂推理任务
- 触发词："请一步步思考"、"Let's think step by step"
- 适用：数学计算、逻辑推理、复杂决策、代码调试

### 少样本学习（Few-shot Learning）
- 提供 2-5 个高质量的示例对（输入-输出配对）
- 示例数量并非越多越好，2-3 个高质量示例通常足够
- 关键：示例需要覆盖不同的输入模式

### 格式控制（Format Control）
- 明确指定输出结构，避免 AI 自由发挥
- 技巧：定义每个字段的含义和约束
- 示例："用 JSON 格式输出，包含 title、summary、body、tags 四个字段"

### 分步骤构建（Step-by-Step Construction）
- 将复杂任务拆分为多个简单的子提示词
- 每个子提示词专注一个方面，最后汇总
- 适用：报告生成、多维度分析、复杂内容创作

### 反迎合提问（Anti-Sycophancy）

> 来自课程 Module 1（1.1）和 Module 2（2.5），吴恩达课程重点强调的 AI 关键缺陷。

**什么是迎合性**：AI 被训练成"取悦用户"，如果你在提示词中透露自己的偏好，AI 很可能会顺着你的话说而非给出客观评价。这是一种由 RLHF（基于人类反馈的强化学习）训练机制导致的系统性偏差。

**影响**：你用 AI 验证自己的想法时，可能得到的是虚假的"同意"而非真实的"分析"。

**反迎合提示技巧**：

| 问题类型 | ❌ 错误写法（诱导迎合） | ✅ 正确写法（反迎合） |
|----------|----------------------|----------------------|
| 评价想法 | "我有一个很棒的商业点子：移动扎染。批评它。" | "请客观分析以下商业理念：移动扎染。使用以下评分标准分析。注意：不要顾及我的感受，我需要真实分析。" |
| 检查文档 | "我写了一份完美的计划，你帮我看一下。" | "请严格审查这份计划。重点关注：逻辑漏洞、数据缺失、风险遗漏。" |
| 询问意见 | "我觉得这种做法最好，你觉得呢？" | "请从效率、成本和风险三个维度比较以下做法的优劣，不需要考虑我的偏好。" |

**核心法则**：想让 AI 说真话，先删除你的"立场提示"。每当你让 AI 评价自己的东西时，先把"我觉得这很好""我做了很多努力"等前置立场语句删掉，换成中性描述。更进一步，可以明确告诉 AI："我需要的是真实分析，哪怕结论是否定的。"

### AI 推理工作流（Reasoning with AI）

> 来自课程 Module 2（2.4），用于需要多步骤逻辑分析的任务。

**适用场景**：方案比较、风险评估、复杂决策、项目规划。

**推理步骤**：
1. 理解任务目标 → 2. 读取上下文材料 → 3. 拆解问题 → 4. 建立评价标准 → 5. 比较备选方案 → 6. 检查限制条件 → 7. 给出建议 → 8. 标注假设和不确定性

**推荐提示词结构**：
```
请认真分析以下问题。不要急着给结论，先：
1. 拆解问题，列出关键假设
2. 建立 [N] 个评价标准及权重
3. 逐项比较每个方案的优劣
4. 给出推荐结论
5. 对不确定的内容单独标注
```

### AI 评审工作流（Critique with AI）

> 来自课程 Module 2（2.7），AI 可以扮演编辑/评委/审稿人，但需要明确的评价标准。

**核心原则**：
- **先找问题再打分**：要求 AI 先逐项找证据，列出具体问题位置，再给综合分数
- **提供评价维度（Rubric）**：将"好不好"拆解为多个可判断维度
- **分块修改优于整篇重写**：逐段评审更可控
- **跨模型评审**：重要材料用多个模型交叉检查，减少单一模型盲点

**推荐 Rubric 维度**：
| 维度 | 关注点 |
|------|--------|
| 主题明确度 | 是否清晰、是否偏题 |
| 结构逻辑 | 段落顺序合理，无逻辑跳跃 |
| 证据支撑 | 有数据、案例或材料支撑观点 |
| 语言表达 | 自然流畅，无模板化套话 |
| 读者体验 | 易于理解，重点突出 |
| 可修改性 | 给出具体可执行的修改动作 |

### 写作工作流（Writing with AI）

> 来自课程 Module 1（1.1）和 Module 2（2.6），核心洞察：先结构，后正文。

**为什么不能直接让 AI 写全文**：
- 大量抽象词，缺乏具体事实
- 段落结构机械，像模板填空
- 观点安全、平均，缺少鲜明判断
- 表面流畅但内容空泛（即"AI 腔" / AI Slop）

**推荐写作流程**：
```
步骤1: 上传背景材料，让 AI 生成 2-3 种不同结构的大纲
步骤2: 人为选择并调整大纲方向
步骤3: 在小标题层面补充要点和素材
步骤4: 确认结构和素材无误后才让 AI 生成正文
步骤5: 逐段评审、修改，减少 AI 腔
```

**消除"AI 腔"技巧**：在提示词中加入"禁用表达"列表，如：
- ❌ "随着时代的发展"
- ❌ "总的来说"
- ❌ "在当今社会"
- ❌ "值得注意的是"

## 提示词优化流程

当用户提供已有提示词时，执行以下诊断优化流程：

### 诊断清单

1. **角色检查** — 是否明确了 AI 的角色？
   - 无角色 → 建议添加角色定义
   - 角色模糊 → 建议具体化
2. **目标检查** — 目标是否具体可衡量？
   - 目标模糊 → 帮助用户明确具体期望
   - 无目标 → 需要补充目的阐述
3. **约束检查** — 是否有足够的约束条件？
   - 约束不足 → 建议添加必要的限制
   - 遗漏关键约束 → 根据场景补充
4. **格式检查** — 输出格式是否明确？
   - 格式模糊 → 建议指定具体格式
   - 格式缺失 → 补充格式要求
5. **冗长检查** — 提示词是否过于冗长？
   - 无关信息过多 → 建议精简
   - 核心信息不够突出 → 建议重组结构

### 优化策略

| 问题类型 | 优化策略 |
|----------|----------|
| 目标不明确 | 用"5W1H"法（What/Why/Who/Where/When/How）帮助定位 |
| 缺少上下文 | 补充项目/任务背景、受众特征、前置条件 |
| 约束过少 | 添加长度、风格、内容、技术等维度约束 |
| 输出不可控 | 指定格式模板、字段定义、结构要求 |
| 缺少示例 | 提供 1-3 个输入-输出示例配对 |

## 场景适配

### 写作创作场景（含写作工作流）
- **核心流程**：大纲 → 批评迭代 → 要点 → 成文（先结构后正文）
- **消除"AI 腔"**：明确禁用表达清单，要求保留具体事实和个人判断
- **风格控制**：指定风格（正式/口语/幽默），提供参考范文
- **内容审核**：用 AI 评审模板检查逻辑跳跃、证据不足、读者理解障碍
- 常见约束：字数范围、语气要求、禁用词列表、输出格式

### 编程开发场景
- 强调：技术栈、性能要求、代码风格、测试需求
- 特殊技巧：提供已有代码片段作为上下文
- 常见约束：编程语言版本、不允许使用某些库、兼容性要求
- **AI 桌面应用**：涉及本地文件操作时，遵循"最小权限→先计划→再确认→后执行"原则

### 数据分析场景
- 强调：数据格式、分析维度、可视化需求、AI 运行代码分析
- **AI 运行代码**：上传数据文件后，AI 可以写代码（Python）并实际运行来计算和绘图
- 特殊技巧：提供数据样例的前 3-5 行、明确分析目标
- 常见约束：分析不包含倾向性、结果需可验证、区分相关性与因果性

### 学习教育场景
- 强调：学习水平、已有知识基础、学习目标
- 特殊技巧：Socratic 提问法（反问引导思考）
- 常见约束：复杂度控制、分步讲解、避免使用特定术语

### 头脑风暴与创意激发
- 强调：多角度产出、迭代深化、角色切换、极端假设
- 特殊技巧：限制激发创意（给反向约束）、"魔鬼代言人"挑战
- 输出处理：实用性/创新性/可执行性标注，筛选最有潜力的方向

### 方案评审与质量控制
- 强调：先找证据再打分、分块评审、Rubric 维度评价
- 特殊技巧：跨模型交叉评审、要求 AI 标注不确定内容
- 适合场景：文章评审、项目方案评审、论文评审、商业计划评审

### 图像生成与编辑
- **图像 Prompt 要素**：主体(Subject) + 场景(Scene) + 动作(Action) + 风格(Style) + 细节(Atmosphere)
- **风格词是"视觉语言"**：掌握越多风格关键词（cinematic / watercolor / cyberpunk 等），控制力越强
- **编辑原则**：明确告诉 AI "保留什么"和"改变什么"，避免过度修改
- **常见局限**：手部结构异常、文字乱码、角色不一致、随机性

### 迷你应用构建
- **核心公式**：明确目标(Goal) + 输入方式(Input) + 输出结果(Output)
- **难度分级**：
  - ✅ 较简单：小游戏、计时器、计算器、单词测验（输入输出明确，边界清楚）
  - ⚠️ 较复杂：联网多人游戏、实时反馈系统（涉及状态同步、网络通信）
- **建议**：从"一个提示词就能表达清楚"的小应用开始，逐步进阶

## 对话示例

### 六步构建示例（简约版）

```
用户: 帮我写个提示词，让 AI 写周报。

AI: 【第一步：角色定义】你希望 AI 扮演什么角色？
用户: 我的工作助理。
AI: 【第二步：背景描述】你的岗位和周报阅读对象？
用户: 后端开发，周报给技术经理看。
AI: 【第三步：目的阐述】希望达到什么效果？
用户: 如实记录，突出技术难点和成果。
AI: 【第四步：约束设定】内容如何组织？
用户: 按"本周完成→下周计划→遇到的问题"三部分，每条简洁。
AI: 【第五步：输出格式】用什么格式？
用户: Markdown。
AI: 【第六步：示例引导】有以前的周报参考吗？（可选）
用户: 没有，按上面要求生成。

→ 生成完整提示词：[角色+背景+输出要求+格式的结构化提示词]
```

### 优化示例

```
用户: 帮我优化这个提示词："写一篇文章"

AI: 诊断结果：
- ❌ 角色未定义 ❌ 缺少上下文 ❌ 目标模糊
- ❌ 无约束条件 ❌ 无格式要求 ❌ 无示例

让我通过六步对话帮你逐步完善...
（逐步收集信息后生成优化版本）
```

## 注意事项

1. 引导用户一步步回答，不要在一步中要求太多信息
2. 对于某步用户无法回答时，提供智能默认值或跳过选项
3. 最终生成的提示词必须是完整的、可直接使用的
4. 如果用户需求复杂，建议拆分为多个子提示词
5. 对于已有提示词的分析，优先使用"少即是多"原则
6. 参考扩展模板见 `reference/prompt_templates.md`
7. 课程知识要点参考见 `reference/course_reference.md`
